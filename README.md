# U-Net Analysis

A repository for analyzing U-Net-like models.

## Development

### Environment
- `uv` >= 0.9.2
- MLflow and PyTorch

### Usage
1. Install dependencies.

    ```bash
    uv sync --group dev
    ```

2. Start training.

    ```bash
    uv run python src/unet_analysis/train
    ```

3. Visualize logs.

    ```bash
    uv run tensorboard --logdir <log dir>
    ```

### Features

#### Datasets

The dataset can include both clean and noisy image pairs.
If a dataset does not provide separate clean and noisy images, noisy images are generated by adding noise.
Each dataset should include index JSON files for splitting the data into training and validation sets.

<details>
<summary>Case 1: paired clean/noisy images</summary>

**Dataset structure:**

- `_mean`: clean image
- `_real`: noisy image

```
  data/dataset1
  |  `-- indices
  |      |-- train_list.json
  |      `-- val_list.json
  |-- xxx_mean.png
  |-- xxx_real.png
  |-- yyy_mean.png
  |-- yyy_real.png
  :
  :
  `-- zzz_real.png
```

</details>

<details>
<summary>Case 2: train/val directories</summary>

**Dataset structure:**

- `_ours`: output image
- `_mean`: clean image
- `_real`: noisy image

```
  data/dataset1
  |  `-- indices
  |      |-- train_list.json
  |      `-- val_list.json
  |-- train
  |   |-- xxx_mean.png
  |   |-- xxx_real.png
  |   |-- yyy_mean.png

  `-- val
```

#### Image Channels

Images can be used in three ways as model input.
- RGB color (3 channels)
- Grayscale (1 channel)
- Dual Channels (2 channels)
  - If grayscale images from two detectors are used, they are fed into the model as two-channel inputs.

#### Weight Analysis

- decay, delta, norm

#### Model Architecture
- CNN-based U-Net
  - Structure
    - ResNet
    - Six stages
    - Skip connections

#### Loss
- Mean Square Error (MSE)
- Mean Absolute Error (MAE)
- SSIM

#### Normalization
- Spectral Normalization
- Batch Normalization

#### MLflow

The following metrics and artifacts are tracked via MLflow:

- **Loss**: training and validation loss per epoch
- **Image statistics**: MSE, SSIM, and other evaluation metrics computed on validation images
- **Weight analysis**: decay, delta, and norm of model weights over training

#### Control Trainable Weights
- Train all parameters
- Train only head parameters
